    1.2 Function (Fungsi dalam PHP)

    Function itu ibarat:

    Mesin kecil yang bisa kamu panggil berkali-kali tanpa menulis ulang kodenya.

    Kalau kamu tidak paham function, nanti:

    Bingung kenapa Controller punya method

    Bingung kenapa ada public function index()

    Bingung kenapa ada parameter di route

    Jadi ini pondasi serius.

    
    
    A. Apa Itu Function?

    Function adalah blok kode yang:

    Bisa dipanggil berulang

    Bisa menerima input

    Bisa mengembalikan output

    Struktur dasar:

    function namaFunction() {
        // kode di sini
    }


    Contoh:

    function sapa() {
        echo "Halo Dunia";
    }

    sapa(); // memanggil function


    Output:

    Halo Dunia

    
    
    B. Function dengan Parameter

    Parameter adalah input untuk function.

    function sapa($nama) {
        echo "Halo " . $nama;
    }

    sapa("Fany");


    Output:

    Halo Fany


    ğŸ“Œ $nama di dalam function disebut parameter
    ğŸ“Œ "Fany" saat dipanggil disebut argument

    
    
    C. Function dengan Return (WAJIB PAHAM)

    Ini yang paling penting.

    Tanpa return:

    function tambah($a, $b) {
        echo $a + $b;
    }

    tambah(5, 3);


    Dengan return:

    function tambah($a, $b) {
        return $a + $b;
    }

    $hasil = tambah(5, 3);
    echo $hasil;


    Kenapa return penting?

    Karena:

    Bisa disimpan ke variabel

    Bisa diproses lagi

    Bisa dipakai di tempat lain

    Laravel sangat bergantung pada return.

    Contoh nanti di controller:

    return view('home');

    
    
    D. Perbedaan echo dan return
    echo	return
    Langsung tampil	Mengirim hasil kembali
    Tidak bisa diproses lagi	Bisa disimpan

    Contoh salah konsep:

    function hitung($a, $b) {
        echo $a + $b;
    }

    $hasil = hitung(5, 3); // hasil jadi NULL


    Kenapa? Karena echo tidak mengembalikan nilai.

    
    
    E. Default Parameter

    Kamu bisa kasih nilai default:

    function sapa($nama = "User") {
        echo "Halo " . $nama;
    }

    sapa();          // Halo User
    sapa("Fany");    // Halo Fany


    Ini sering dipakai di Laravel.

    
    
    F. Scope (Ruang Lingkup Variabel)

    Ini sering bikin error pemula.

    function test() {
        echo $nama; // ERROR
    }


    Kenapa error?

    Karena variabel di luar function tidak otomatis bisa dipakai di dalam.

    Solusi 1 (tidak disarankan):

    function test() {
        global $nama;
        echo $nama;
    }


    Solusi benar:

    function test($nama) {
        echo $nama;
    }

    test("Fany");


    ğŸ“Œ Biasakan kirim data lewat parameter, bukan pakai global.

    1.4 OOP PHP (Object Oriented Programming)
    
    
    
    A. Apa Itu OOP?

    OOP adalah cara menulis program dengan konsep:

    Object (Objek) yang dibuat dari blueprint bernama Class.

    Bayangkan seperti ini:

    Class = Cetakan / Blueprint rumah

    Object = Rumah yang sudah jadi

    
    
    
    B. Class & Object
    1ï¸âƒ£ Membuat Class
    class User {
        public $nama;
    }


    Ini baru blueprint.

    2ï¸âƒ£ Membuat Object
    $user1 = new User();


    new artinya membuat object dari class.

    3ï¸âƒ£ Mengisi Property
    $user1->nama = "Fany";
    echo $user1->nama;


    Output:

    Fany


    ğŸ“Œ Tanda -> artinya mengakses isi object.

    Ini nanti sangat sering muncul di Laravel:

    $user->email
    $post->title
    $request->input()


    Semua itu object.

    
    
    
    C. Method (Function di dalam Class)

    Function di dalam class disebut method.

    class User {
        public $nama;

        public function sapa() {
            return "Halo " . $this->nama;
        }
    }


    Gunakan $this untuk mengakses property dalam class.

    Contoh lengkap:

    $user1 = new User();
    $user1->nama = "Fany";

    echo $user1->sapa();


    Output:

    Halo Fany


    ğŸ“Œ $this = object itu sendiri.

    
    
    
    D. Constructor

    Constructor adalah function khusus yang otomatis jalan saat object dibuat.

    Namanya selalu:

    __construct()


    Contoh:

    class User {
        public $nama;

        public function __construct($nama) {
            $this->nama = $nama;
        }

        public function sapa() {
            return "Halo " . $this->nama;
        }
    }


    Pemakaian:

    $user1 = new User("Fany");
    echo $user1->sapa();


    Constructor membuat object langsung punya data.

    Laravel sangat sering pakai ini (Dependency Injection nanti).

    
    
    
    E. Visibility (public / private / protected)

    Ini tentang siapa yang boleh mengakses property atau method.

    1ï¸âƒ£ Public

    Bisa diakses dari mana saja.

    public $nama;

    2ï¸âƒ£ Private

    Hanya bisa diakses di dalam class itu sendiri.

    private $password;


    Tidak bisa:

    $user->password; // ERROR

    3ï¸âƒ£ Protected

    Bisa diakses oleh class itu dan turunannya (inheritance).

    
    
    
    F. Inheritance (Pewarisan)

    Class bisa mewarisi class lain.

    class User {
        public $nama;

        public function sapa() {
            return "Halo " . $this->nama;
        }
    }

    class Admin extends User {
        public function akses() {
            return "Akses penuh";
        }
    }


    Sekarang:

    $admin = new Admin();
    $admin->nama = "Fany";

    echo $admin->sapa();  // dari User
    echo $admin->akses(); // dari Admin


    Admin mewarisi semua milik User.

    Laravel banyak memakai inheritance:

    Controller mewarisi base Controller

    Model mewarisi Model bawaan Laravel

    
    
    
    G. Kenapa OOP Ini Penting untuk Laravel?

    Nanti kamu akan lihat:

    class UserController extends Controller


    Itu inheritance.

    Atau:

    public function __construct(Request $request)


    Itu constructor.

    Atau:

    $user->email


    Itu property object.

    Atau:

    User::where(...)


    Itu static method (nanti level lanjut).

    Saling terhubung

    Saling dipanggil

    Bukan kumpulan file random.
