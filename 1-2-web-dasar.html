    A. Apa Itu HTTP?

    HTTP adalah:

    Bahasa komunikasi antara Browser dan Server.

    2.2 GET vs POST

    (Bedah Dalam + Keamanan + Kapan Pakai Apa)

    
    
    
    
    A. Apa Itu GET?

    GET adalah method HTTP untuk:

    Mengambil data dari server.

    Contoh saat kamu buka halaman:

    GET /products/5


    Contoh URL dengan parameter:

    /search?keyword=tas&harga=100000


    Data dikirim lewat URL.

    Ciri-Ciri GET

    âœ” Data terlihat di URL
    âœ” Bisa di-bookmark
    âœ” Bisa di-share
    âœ” Biasanya untuk mengambil data
    âœ” Tidak punya body (umumnya)

    Contoh GET di PHP
    echo $_GET['keyword'];


    Kalau URL:

    domain.com/search?keyword=tas


    Maka output:

    tas

    
    
    
    
    B. Apa Itu POST?

    POST adalah method HTTP untuk:

    Mengirim data ke server untuk diproses.

    Biasanya dipakai untuk:

    Login

    Register

    Simpan data

    Upload file

    Data dikirim lewat body, bukan URL.

    Ciri-Ciri POST

    âœ” Data tidak terlihat di URL
    âœ” Tidak bisa di-bookmark
    âœ” Biasanya untuk mengubah data
    âœ” Lebih aman dibanding GET (untuk data sensitif)

    Contoh POST di PHP
    echo $_POST['email'];


    Biasanya berasal dari form:

    


    
    
    
    
        C. Perbandingan GET vs POST
    GET	POST
    Ambil data	Kirim data
    Data di URL	Data di body
    Bisa di-cache	Tidak di-cache
    Kurang aman	Lebih aman
    Idempotent (tidak ubah data)	Bisa mengubah data
    
    
    
    
    D. Keamanan GET vs POST

    Penting banget ini.

    âŒ Jangan pakai GET untuk:

    Password

    Data sensitif

    Transaksi

    Update database

    Karena:

    Bisa terlihat di URL

    Tersimpan di history browser

    Bisa disalin orang

    âœ” Gunakan POST untuk:

    Login

    Register

    Pembayaran

    Edit data

    Upload file

    
    
    
    
    E. Kenapa GET Tidak Boleh Mengubah Data?

    GET harusnya hanya membaca data.

    Kalau kamu buat:

    GET /delete/5


    Itu bahaya.

    Karena:

    Bisa diakses langsung lewat URL

    Bisa terindeks Google

    Bisa di-trigger tanpa sengaja

    Makanya di Laravel:

    Route::delete(...)
    Route::post(...)


    Untuk operasi yang mengubah data.

    
    
    
    
    F. Konsep Idempotent (Sedikit Advanced Tapi Penting)

    Idempotent artinya:

    Kalau request dilakukan berkali-kali, hasilnya tetap sama.

    GET bersifat idempotent.

    Contoh:

    GET produk 5

    GET produk 5 lagi

    Tidak mengubah apa-apa

    POST tidak selalu idempotent.

    Contoh:

    POST create order

    Dikirim 2x

    Bisa bikin 2 order

    Ini penting banget untuk sistem pembayaran.

    
    
    
    
    G. GET dan POST di Laravel

    Contoh route:

    Route::get('/products', [ProductController::class, 'index']);
    Route::post('/products', [ProductController::class, 'store']);


    GET â†’ tampilkan data
    POST â†’ simpan data

    Laravel sangat mengikuti standar HTTP.

    
    
    
    
    H. Kenapa Laravel Pakai CSRF di POST?

    Karena POST bisa mengubah data.

    Laravel otomatis memberi proteksi:

    @csrf


    Untuk mencegah:

    CSRF Attack

    Form disalahgunakan

    Ini tidak diperlukan untuk GET karena GET tidak boleh mengubah data.

    
    
    
    
    I. Kapan Pakai Apa? (Rule Sederhana)

    Gunakan GET jika:

    Hanya membaca data

    Tidak mengubah database

    Bisa diakses publik

    Gunakan POST jika:

    Mengirim form

    Mengubah data

    Login / register

    Transaksi

    2.3 Session & Cookie

    (Kenapa Login Bisa Tetap Aktif?)

    
    
    
    A. Masalah Besarnya: HTTP Itu Stateless

    Kita ulang sedikit:

    Setiap request ke server itu berdiri sendiri.

    Artinya:

    Kamu login â†’ server proses

    Kamu buka dashboard â†’ itu request baru

    Server sebenarnya tidak otomatis tahu itu orang yang sama

    Kalau tidak ada sistem tambahan, maka:

    Setiap refresh = dianggap user baru

    Tidak ada konsep â€œsudah loginâ€

    Maka dibuatlah:

    Session dan Cookie

    
    
    
    B. Apa Itu Cookie?

    Cookie adalah:

    Data kecil yang disimpan di browser user.

    Contoh isi cookie:

    Session ID

    Token login

    Preferensi bahasa

    Remember me

    Cara kerjanya:

    Server kirim cookie ke browser

    Browser menyimpan

    Setiap request berikutnya, browser kirim cookie itu lagi

    Cookie disimpan di sisi client (browser).

    
    
    
    C. Apa Itu Session?

    Session adalah:

    Data yang disimpan di server untuk mengingat user.

    Biasanya yang disimpan:

    user_id

    status login

    role

    data sementara

    Session disimpan di:

    File

    Database

    Redis

    Memory

    Tapi browser tidak tahu isi session.
    Browser hanya tahu ID-nya saja.

    
    
    
    D. Cara Kerja Login Secara Teknis

    Ini alur sebenarnya waktu kamu login:

    1ï¸âƒ£ User kirim POST login
    POST /login
    email & password

    2ï¸âƒ£ Server cek database

    Kalau benar:

    Server buat session:

    session:
    user_id = 5

    3ï¸âƒ£ Server kirim cookie berisi Session ID

    Misalnya:

    Set-Cookie: session_id=abc123

    4ï¸âƒ£ Browser simpan cookie

    Sekarang browser punya:

    session_id = abc123

    5ï¸âƒ£ User buka dashboard

    Browser otomatis kirim:

    Cookie: session_id=abc123


    Server:

    Cari session dengan ID abc123

    Ketemu user_id = 5

    Tahu ini user yang sudah login

    Dashboard bisa ditampilkan.

    
    
    
    E. Ringkasan Perbedaan Session vs Cookie
    Session	Cookie
    Disimpan di server	Disimpan di browser
    Lebih aman	Bisa dilihat user
    Menyimpan data login	Biasanya menyimpan ID session

    Laravel secara default menggunakan:

    Session untuk menyimpan data login

    Cookie untuk menyimpan session ID

    
    
    
    F. Kenapa Cookie Saja Tidak Cukup?

    Karena cookie:

    Bisa dimodifikasi user

    Tidak aman untuk menyimpan password

    Makanya:
    Cookie hanya menyimpan ID,
    Data aslinya disimpan di session server.

    
    
    
    G. Session di PHP Dasar

    Contoh sederhana:

    session_start();

    $_SESSION['nama'] = "Fany";


    Ambil session:

    echo $_SESSION['nama'];


    Kalau session tidak ada:

    if (!isset($_SESSION['nama'])) {
        echo "Belum login";
    }


    Laravel mengatur ini lebih rapi lewat middleware auth.

    
    
    
    H. Konsep â€œRemember Meâ€

    Kalau kamu centang â€œremember meâ€:

    Cookie disimpan lebih lama

    Session bisa dibuat persistent

    User tidak logout setelah browser ditutup

    Tanpa itu:

    Session biasanya hilang saat browser ditutup

    
    
    
    I. Kenapa Session Penting untuk Laravel?

    Nanti kamu akan lihat:

    Auth::user();


    Itu sebenarnya membaca data dari session.

    Atau:

    session(['cart' => $data]);


    Laravel menyimpan data di session.

    
    
    
    J. Ancaman Keamanan (Sedikit Awareness)

    Beberapa ancaman:

    Session Hijacking

    CSRF

    Cookie tampering

    Makanya Laravel punya:

    CSRF protection

    Encrypted cookies

    Regenerate session ID

    A. Apa Itu Form?
