3. Database (Konsep Dasar Dulu)
    
    
    
    
    
    A. Apa Itu Database?

    Database adalah:

    Tempat menyimpan data secara terstruktur agar bisa dicari, diubah, dan dihapus dengan mudah.

    Contoh data dalam aplikasi:

    User

    Produk

    Pesanan

    Artikel

    Transaksi

    Role & Permission

    Tanpa database:

    Data hilang saat server mati

    Tidak bisa login permanen

    Tidak bisa menyimpan order

    
    
    
    
    
    B. Kenapa Database Dibutuhkan?

    Bayangkan kamu buat sistem login tanpa database.

    User daftar â†’ lalu disimpan di variabel.

    Begitu server restart â†’ semua hilang.

    Database menyimpan data secara persisten.

    Artinya:

    Data tetap ada walaupun server mati.

    
    
    
    
    
    C. Struktur Mental Database

    Database punya struktur bertingkat:

    Database
    â†“
    Table
    â†“
    Row (Record)
    â†“
    Column (Field)


    Contoh sistem toko online:

    Database: tokoku

    Table:

    users

    products

    orders

    Table users:

    id	nama	email	password

    Column = id, nama, email

    Row = 1 baris user

    
    
    
    
    
    D. Database Itu Seperti Excel Tapi Lebih Pintar

    Kalau Excel:

    Bisa simpan tabel

    Bisa filter

    Bisa hitung

    Database:

    Bisa relasi antar tabel

    Bisa jutaan data

    Bisa dipakai banyak user bersamaan

    Bisa diakses program (Laravel)

    
    
    
    
    
    E. Relasi Antar Data (Konsep Paling Penting)

    Di dunia nyata, data tidak berdiri sendiri.

    Contoh:

    User membeli Produk.

    Berarti:

    1 user bisa punya banyak order

    1 order punya banyak produk

    Inilah yang disebut relasi.

    Tanpa relasi:

    Data berantakan

    Banyak duplikasi

    Sulit dikontrol

    
    
    
    
    
    F. Kenapa Tidak Simpan Semua dalam 1 Table?

    Misalnya kamu buat:

    | user_nama | user_email | product_nama | harga |

    Masalah:

    User beli 5 produk â†’ data user diulang 5 kali

    Tidak efisien

    Sulit update

    Maka data dipisah:

    Table users
    Table products
    Table orders

    Dan dihubungkan dengan ID.

    
    
    
    
    
    G. Primary Key (Identitas Unik)

    Primary key adalah:

    Penanda unik setiap baris data.

    Biasanya:

    id


    Contoh:

    id	nama
    1	Fany
    2	Budi

    ID tidak boleh sama.

    Laravel default selalu pakai id.

    
    
    
    
    
    H. Foreign Key (Penghubung Relasi)

    Foreign key adalah:

    ID dari tabel lain yang dipakai untuk menghubungkan data.

    Contoh:

    Table orders:

    id	user_id	total
    1	2	50000

    user_id menunjuk ke:

    users.id = 2

    Itu artinya:
    Order ini milik Budi.

    
    
    
    
    
    I. Jenis Relasi (Pondasi untuk Laravel Eloquent)

    1 user â†’ 1 profile

    2ï¸âƒ£ One to Many

    1 user â†’ banyak order

    3ï¸âƒ£ Many to Many

    1 produk â†’ banyak kategori

    J. Konsep CRUD (Sangat Penting)

    Semua aplikasi database hanya melakukan 4 hal:

    C â†’ Create (Simpan)

    R â†’ Read (Ambil)

    U â†’ Update (Ubah)

    3.1 MySQL

    (Apa itu MySQL + cara kerja + kenapa dipakai Laravel)

    
    
    
    
    
    A. Apa Itu MySQL?

    MySQL adalah:

    Software Database Management System (DBMS) yang menggunakan SQL untuk mengelola data.

    Artinya:

    MySQL = mesin pengelola database

    SQL = bahasa untuk berbicara dengan database

    Bayangin seperti ini:

    Kamu = Developer

    SQL = Bahasa

    MySQL = Mesin yang menjalankan perintah

    MySQL adalah sistem database open-source paling populer di dunia web.

    Laravel secara default menggunakan MySQL.

    
    
    
    
    
    B. Kenapa MySQL Sangat Populer?

    Beberapa alasan:

    âœ… Gratis & open-source

    PHP dan MySQL itu kombinasi klasik di dunia web.

    Laravel sebagai framework PHP otomatis sangat cocok dengan MySQL.

    
    
    
    
    
    C. Cara Kerja MySQL (Gambaran Besar)

    Alurnya seperti ini:

    Laravel
    â†“
    Query SQL
    â†“

    â†“
    Database

    A. Apa Itu Desain Database?

    Desain database adalah:

    D. Primary Key (Identitas Utama)

    Primary key adalah:

    Biasanya:

    Sifatnya:

    Artinya:

    | id | nama |

    G. Normalisasi (Konsep Anti Berantakan)

    Normalisasi artinya:

    Mengatur struktur tabel agar tidak ada data berulang yang tidak perlu.

    Contoh buruk:
    Simpan alamat user di setiap order.

    Lebih baik:
    Simpan alamat di table users.

    Semakin rapi struktur,

    3.3 Basic Query SQL (SELECT, INSERT, UPDATE, DELETE)
