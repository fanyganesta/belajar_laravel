    D. Cara Install Laravel

    Lalu:

    Tapi untuk pemula, pakai cara pertama saja.

    C. Struktur Umum composer.json

    Biasanya di Laravel ada bagian seperti ini:

    Saat kamu menginstal Laravel versi terbaru (saat ini Laravel 11), prosesnya jauh lebih ramping dan cepat dibanding versi-versi sebelumnya. Laravel sekarang menggunakan "Skeleton" yang sangat minimalis.

    Namun, ada beberapa poin konfirmasi penting yang akan muncul di terminal saat kamu menjalankan perintah laravel new nama-proyek atau composer create-project:

    
    
    
    
    1. Pilihan Starter Kit (Breeze / Jetstream)
    Laravel akan bertanya apakah kamu ingin menginstal Starter Kit.

    None (No starter kit): Cocok kalau kamu mau bangun semuanya dari nol.

    Breeze: Sangat direkomendasikan untuk pemula/menengah. Ini memberikan sistem Login, Register, dan Profil yang simpel (pilihannya ada Blade, Livewire, atau Vue/React via Inertia).

    Jetstream: Untuk aplikasi yang lebih kompleks (ada fitur Team, Two-Factor Auth, dll).

    
    
    
    
    2. Pilihan Testing Framework
    Kamu akan diminta memilih antara:

    Pest: Framework testing baru yang sedang naik daun karena sintaksnya sangat manusiawi dan simpel.

    PHPUnit: Standar lama yang sudah sangat stabil.

    Saran: Kalau kamu baru mulai, Pest sangat menyenangkan untuk dipelajari.

    
    
    
    
    3. Inisialisasi Git Repository
    Laravel akan menawarkan untuk otomatis menjalankan git init. Jika kamu berencana menyimpan kode di GitHub/GitLab, pilih Yes. Ini akan otomatis membuatkan file .gitignore sehingga file sampah (seperti folder node_modules atau file .env) tidak ikut terunggah.

    
    
    
    
    4. Konfirmasi Database (Sangat Penting!)
    Di versi terbaru, Laravel lebih proaktif. Setelah file terinstal, ia akan bertanya:

    "Which database database would you like to use?" (SQLite, MySQL, PostgreSQL, dll).

    Jika kamu memilih SQLite (default sekarang untuk pengembangan lokal), Laravel akan langsung membuatkan file database-nya untukmu. Jika kamu memilih MySQL, pastikan server database kamu (XAMPP/Laragon) sudah menyala.

    Hal Lain yang Harus Kamu Cek Manual:
    Setelah instalasi selesai, ada beberapa hal "di balik layar" yang harus kamu pastikan:

    Versi PHP: Laravel 11 membutuhkan minimal PHP 8.2. Jika versi PHP kamu di bawah itu, instalasi akan gagal.

    Application Key: Pastikan di file .env kamu sudah ada APP_KEY. Jika belum, jalankan php artisan key:generate. Tanpa ini, enkripsi (termasuk Cookie dan CSRF yang kita bahas tadi) tidak akan jalan.

    Folder Permissions: Jika kamu menggunakan Linux/Mac, pastikan folder storage dan bootstrap/cache bisa ditulis oleh server (chmod -R 775).

    Perubahan Struktur di Versi Terbaru:
    Jangan kaget kalau kamu tidak menemukan folder app/Http/Kernel.php atau banyak file di folder config/. Laravel terbaru memindahkan konfigurasi-konfigurasi tersebut ke dalam file bootstrap/app.php agar struktur folder terlihat lebih bersih.

    Tips: Selalu jalankan php artisan serve setelah instalasi untuk memastikan halaman "Welcome" muncul. Jika muncul, berarti instalasi sukses!
