    A. Apa Itu File .env?

    .env adalah:

    File konfigurasi environment untuk aplikasi Laravel.

    Isinya biasanya:

    APP_NAME=Laravel
    APP_ENV=local
    APP_KEY=base64:xxxx
    APP_DEBUG=true
    APP_URL=http://localhost

    DB_CONNECTION=mysql
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=laravel
    DB_USERNAME=root
    DB_PASSWORD=


    File ini mengatur:

    Mode aplikasi

    Database

    URL

    Email

    Cache

    Queue

    dan banyak lagi

    
    
    
    
    
    B. Kenapa Disebut ‚ÄúEnvironment‚Äù?

    Karena aplikasi bisa berjalan di beberapa lingkungan:

    local (komputer sendiri)

    staging (server testing)

    production (server live)

    Konfigurasinya bisa beda.

    Contoh:

    Local	Production
    DB lokal	DB server
    Debug aktif	Debug mati

    Itu sebabnya .env dibuat terpisah.

    D. Bagaimana Laravel Membaca .env?

    Laravel akan membaca .env saat booting.

    Nilainya bisa dipanggil di config:

    env('APP_NAME')


    Tapi best practice:
    Gunakan config file, bukan langsung env() di controller.

    
    
    
    
    
    E. Kenapa .env Tidak Boleh Masuk GitHub?

    Karena di dalamnya ada:

    ‚ùå Password database
    ‚ùå API key
    ‚ùå Secret key
    ‚ùå Email credential

    F. Apa Itu .env.example?

    Laravel menyediakan:

    .env.example


    Ini template untuk environment.

    Kalau clone project:

    cp .env.example .env
    php artisan key:generate

    
    
    
    
    
    G. Kalau Ubah .env Tapi Tidak Berubah?

    Laravel menyimpan cache config.

    Kalau kamu ubah .env tapi tidak efek, jalankan:

    php artisan config:clear


    Atau:

    php artisan config:cache
