    ADDITIONAL
    Perbedaan paling mendasar antara keduanya adalah "di mana logika (otak) komponen itu berada."

    Jika Anonymous Component hanya punya satu file (Blade saja), maka Class-based Component (Komponen berbasis Class) punya dua file yang bekerja berpasangan:

    1. File Class (PHP): Berlokasi di `app/View/Components/NamaKomponen.php`. Ini adalah otaknya.
    2. File Template (Blade): Berlokasi di `resources/views/components/nama-komponen.blade.php`. Ini adalah tampilannya.

    Berikut adalah tabel perbandingannya agar lebih mudah dibaca:

    ### Tabel Perbandingan

    | Fitur | Anonymous Component | Class-based Component |
    | --- | --- | --- |
    | Jumlah File | 1 File (`.blade.php`) | 2 File (`.php` & `.blade.php`) |
    | Lokasi Logika | Langsung di dalam Blade | Di dalam method `render` atau `__construct` di Class PHP |
    | Cara Buat | Manual buat file di folder components | Jalankan `php artisan make:component Nama` |
    | Complex Query | Kurang cocok (Blade jadi kotor) | Sangat cocok (Bisa ambil data DB di Class) |
    | Kapan Pakai? | UI Simpel (Button, Alert, Input) | UI Kompleks (Sidebar Dinamis, Dashboard Stat, Tabel) |

    ### Kapan Kamu "Wajib" Menggunakan Class-based Component?

    Ada situasi di mana Anonymous Component sudah tidak sanggup lagi atau membuat kodemu berantakan:

    #### 1. Ketika Butuh Logika PHP yang Rumit

    Bayangkan kamu punya komponen "Buku Terpopuler". Komponen ini harus mengambil 5 buku dengan rating tertinggi dari database.

    * Anonymous: Kamu harus melakukan query di dalam Blade (sangat tidak disarankan dan bikin pusing).
    * Class-based: Kamu lakukan query di dalam Class PHP, lalu kirim hasilnya ke Blade.

    #### 2. Ketika Ingin Memanipulasi Data Sebelum Tampil

    Misalkan kamu mengirim data `tanggal_lahir`, tapi kamu ingin komponen menampilkannya sebagai `umur`.

    * Di Class PHP: Kamu bisa buat fungsi `hitungUmur()` di dalam Class, lalu di Blade tinggal panggil `{{ $umur }}`.

    #### 3. Menghindari Pengulangan Kode (Dry)

    Jika ada logika yang sama digunakan oleh banyak komponen, kamu bisa menggunakan teknik Inheritance (Pewarisan) pada Class PHP-nya.

    Class-based Component (`app/View/Components/Alert.php`):

    ### Kesimpulan Sederhana:

    * Pilih Anonymous: Jika kamu hanya ingin memindahkan kode HTML yang panjang ke file terpisah agar file utama bersih. Ibaratnya cuma "pindah lemari".
    * Pilih Class-based: Jika komponenmu butuh "berpikir" sendiri sebelum menampilkan sesuatu. Ibaratnya komponen tersebut punya "otak kecil" sendiri.

    Biasanya, saat baru mulai belajar Laravel, kamu akan 90% menggunakan Anonymous Component karena lebih praktis. Kamu baru akan butuh Class-based saat aplikasimu sudah besar dan butuh pengolahan data yang rapi di sisi backend.
